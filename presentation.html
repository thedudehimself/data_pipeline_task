Deconstructing the Problem & The Initial Strategy

The Core Request

  • Clean 500,000+ HTML reviews.
  • Categorize ~74,000 unique products without a lookup table.

The Constraints (from Richard)

  • No API Access: The most reliable method was off the table.
  • 1-Day Timeframe: Speed of delivery was critical.
  • >80% Accuracy Goal, 90%+ a Stretch Goal: The final result needed to be trustworthy.

My Recommended Strategy: A Hybrid Approach

Based on the constraints, I proposed a Web Scraping + NLP hybrid. The plan was to scrape a small "ground truth" dataset, then use that labelled data to train a machine learning model that could predict the rest.

This seemed like the fastest path to a complete, accurate solution.

The Journey Pt. 1: Foundational ETL & Data Discovery

  • Step 0: Data-First Approach (`step_0_data_discovery.py`)
    Before writing any processing code, I started by analyzing the raw data. This immediately revealed the scale of the problem and the high percentage of messy HTML content, confirming the need for a robust cleaning pipeline.
  • Step 1: Building a Scalable Cleaning Pipeline (`step_1_elt.py`)
    I built a script using Python and Pandas to process the data in memory-efficient chunks, ensuring it could handle a file of any size without crashing. The script successfully stripped all HTML, producing a clean `Cleaned_Reviews.csv`.
(Talking Point): "I believe in a 'measure twice, cut once' approach. The initial discovery step was crucial because it informed the design of the entire pipeline and prevented future surprises. Identifing and mitigating the blockers to success before building is key."

The Journey Pt. 2: The "Aha!" Moment

The Problem: The first NLP model produced a 92% accuracy score, but the output was clearly wrong—it predicted "Grocery & Gourmet Food" for almost everything.

The Diagnosis: I didn't trust the misleading metric. I wrote a diagnostic script to investigate the training data we had scraped.

The Insight: The analysis proved that our training data was over 90% food-related. The model wasn't broken; it was just perfectly reflecting the heavy bias in the data it was given.

(Talking Point): "This was the most critical pivot point. The data showed that continuing down the initial path would mean delivering a complete, but fundamentally flawed and untrustworthy, dataset. As an engineer, my responsibility is to deliver value, and a model that's always wrong isn't valuable."

The Journey Pt. 3: The Pivot to Intelligent Data Acquisition

  • The New Strategy: The diagnosis showed that random sampling for training data was a failing strategy. The solution was to stop scraping randomly and start scraping strategically.
  • The Implementation (`step_2_webscraping...py`):
    • I re-engineered the scraping phase to use a targeted acquisition approach.
    • The script now pre-scans all 500,000 reviews for keywords related to our under-represented categories (like "dog," "shampoo," "toy").
    • This creates a prioritized "shopping list" of products to scrape, guaranteeing a more balanced and intelligent training dataset for the model.

The entire project is packaged in a clean, professional Git repository.
